Kubeadm Init Installation Workflow

Start: kubeadm init Command
⬇
Preflight Checks
CPU, Memory, Ports, Container Runtime
⬇
Generate TLS Certificates
(API Server, Scheduler, etcd)
⬇
Create kubeconfig Files
admin.conf, scheduler.conf
⬇
Create Static Pod Manifests
⬇
kubelet Starts Control Plane Containers
⬇
API Server Starts Listening on Port 6443
⬇
Upload Cluster Configuration
⬇
Control Plane Node Initialized
⬇
Generate Worker Join Token
⬇
Install CNI Network Plugin


Important kubeadm init Switches

Switch Description Example
--apiserver-advertise-address IP address the Kubernetes API Server will advertise to other nodes kubeadm init --apiserver-advertise-address=192.168.1.10
--pod-network-cidr Defines the Pod network range required by CNI plugins like Calico or Flannel kubeadm init --pod-network-cidr=192.168.0.0/16
--service-cidr Defines the IP address range used by Kubernetes Services kubeadm init --service-cidr=10.96.0.0/12
--control-plane-endpoint Stable endpoint (DNS or Load Balancer) used for HA control plane setups kubeadm init --control-plane-endpoint=k8s-lb.company.com
--kubernetes-version Specify which Kubernetes version to install kubeadm init --kubernetes-version=v1.30.0
--node-name Specify the node name for the control plane kubeadm init --node-name=master-node
--cri-socket Specify container runtime socket (containerd or CRI-O) kubeadm init --cri-socket=/run/containerd/containerd.sock
--upload-certs Uploads control plane certificates so additional master nodes can join kubeadm init --upload-certs
--ignore-preflight-errors Allows kubeadm to ignore specific environment validation errors kubeadm init --ignore-preflight-errors=Swap
--config Uses a YAML configuration file instead of command line parameters kubeadm init --config=kubeadm-config.yaml


Example Custom Installation Command


kubeadm init \
--apiserver-advertise-address=192.168.1.10 \
--pod-network-cidr=192.168.0.0/16 \
--service-cidr=10.96.0.0/12 \
--kubernetes-version=v1.30.0
