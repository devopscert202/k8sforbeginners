Pre-Flight

Prepare Your Environment First

Project Setup

Update the script variables

Replace placeholder values like your-project and make sure the script variables point to your real project, cluster name, region, and key file.

Authentication

Confirm login or key-based auth

You can use interactive login for a user session or use a service-account key when the scripts should run without manual prompts.

CLI Tools

Verify gcloud and kubectl

The create script checks for both commands. Install them and make sure they are available from the PowerShell session you use.

File Placement

Keep the key with the scripts

The scripts resolve the key file relative to their own folder, so it is easiest to keep the JSON key in the same directory.

Cluster Build

Create The GKE Cluster

1

Open PowerShell in the script folder

Run the scripts from the directory that contains the PowerShell files and your JSON key file.

2

Validate access and tools

Make sure your credentials, project context, gcloud, and kubectl are all ready before cluster creation.

3

Run the create script

The script checks prerequisites, enables APIs, creates the network if needed, then creates the cluster.

4

Fetch credentials and verify nodes

The script finishes by pulling kubeconfig credentials and showing the node list with kubectl.

Run The Script
powershell -ExecutionPolicy Bypass -File .\create-gke-k8sdemo.ps1
What The Script Does
gcloud auth activate-service-account --key-file="your-key.json"
gcloud config set project your-project
gcloud services enable container.googleapis.com compute.googleapis.com iam.googleapis.com serviceusage.googleapis.com
gcloud compute networks create default --project=your-project --subnet-mode=custom
gcloud compute networks subnets create default-asia-south1 --project=your-project --network=default --region=asia-south1 --range=10.10.0.0/20 --secondary-range="gke-pods=10.20.0.0/16,gke-services=10.30.0.0/20"
gcloud container clusters create gcp-k8sdemo --project=your-project --region=asia-south1 --node-locations=asia-south1-a,asia-south1-b,asia-south1-c --network=default --subnetwork=default-asia-south1 --cluster-secondary-range-name=gke-pods --services-secondary-range-name=gke-services --enable-ip-alias --num-nodes=1 --machine-type=e2-medium --release-channel=regular
gcloud container clusters get-credentials gcp-k8sdemo --region=asia-south1 --project=your-project
kubectl get nodes -o wide
This cluster is regional, so --num-nodes=1 means one node in each selected zone, for three nodes total.
Node Pool

Resize The Cluster

When To Use

After the cluster already exists

Use the resize script when you want to change the node count for the default node pool without recreating the cluster.

Current Target

1 node per zone

The current resize script sets the regional default pool to one node in each zone.

Run The Script
powershell -ExecutionPolicy Bypass -File .\resize-gke-k8sdemo.ps1
Manual Command
gcloud auth activate-service-account --key-file="your-key.json"
gcloud config set project your-project
gcloud container clusters resize gcp-k8sdemo --project=your-project --region=asia-south1 --node-pool=default-pool --num-nodes=1 --quiet
Cluster Removal

Delete Only The Cluster

Confirmation

Delete asks for safety input

The delete script prompts you to type DELETE before it removes the cluster.

Scope

Cluster only

This script removes the GKE cluster and leaves project IAM access unchanged.

Run The Script
powershell -ExecutionPolicy Bypass -File .\delete-gke-k8sdemo.ps1
Manual Command
gcloud auth activate-service-account --key-file="your-key.json"
gcloud config set project your-project
gcloud container clusters delete gcp-k8sdemo --project=your-project --region=asia-south1 --quiet
Full Cleanup

Teardown Cluster And Access

1

Confirm intent

The wrapper asks you to type TEARDOWN so you do not accidentally remove both resources and access.

2

Delete the cluster

The cluster is removed first from the configured region.

3

Revoke editor access

The script then removes the roles/editor binding for wayfairtest521@gmail.com.

Run The Wrapper
powershell -ExecutionPolicy Bypass -File .\teardown.ps1
Manual Sequence
gcloud auth activate-service-account --key-file="your-key.json"
gcloud config set project your-project
gcloud container clusters delete gcp-k8sdemo --project=your-project --region=asia-south1 --quiet
gcloud projects remove-iam-policy-binding your-project --member="user:wayfairtest521@gmail.com" --role="roles/editor"
Use the teardown wrapper when you want one command flow for cluster cleanup and access cleanup together.
