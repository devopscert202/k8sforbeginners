# Kubernetes upgrade: v1.31 → v1.32

For upgrade prerequisites, version skew policy, etcd backup, drain/cordon order, validation, and generic rollback concepts, see **[Upgrade procedure](./upgrade_procedure.md)**.

This guide focuses on **v1.32** release highlights, API notes, and **command examples** for **`kubeadm` / `kubelet` / `kubectl` at 1.32.0** (Debian/Ubuntu and RHEL/CentOS-style snippets). Adjust package suffixes (`-00`, `-0`, etc.) to match what your repository publishes (`apt-cache madison` / `yum list`).

---

## Overview

Upgrade **one minor at a time** from **v1.31** to **v1.32**: control plane first ( **`kubeadm upgrade apply`** on the first control plane node, **`kubeadm upgrade node`** on additional control plane members), then workers. See the [upgrade procedure](./upgrade_procedure.md) for HA ordering and skew rules.

---

## Prerequisites (v1.32-specific)

- Cluster running Kubernetes **v1.31**
- Read [Kubernetes v1.32 release notes / changelog](https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG/CHANGELOG-1.32.md)
- Confirm workloads and controllers are compatible with **v1.32**
- Scan for **deprecated API** usage before upgrading (commands below)

For etcd backup, staging clusters, and drain flags, use **[Upgrade procedure](./upgrade_procedure.md)**.

---

## What's new in v1.32

### Major features

1. **Gateway API v1.2 GA** — advanced traffic management capabilities
2. **Pod scheduling improvements** — scheduler performance work
3. **Image pull policy enhancements** — finer-grained image pull behavior
4. **Kubeadm improvements** — smoother upgrade UX
5. **Security enhancements** — authentication and authorization hardening

### API changes

#### Graduated to stable (GA)

- `gateway.networking.k8s.io/v1` — Gateway API core resources
- `ClusterTrustBundle` API — certificate trust bundle management

#### Deprecations

- None called out here for core resources; verify against the changelog for your workloads.

#### Removed APIs

- No major removals called out here; confirm with the official changelog before upgrade.

### Important notes for this minor

1. **etcd:** Recommended **etcd v3.5.15+** (confirm against Kubernetes release notes for your exact patch).
2. **Container runtime:** **containerd 1.7.x** or **CRI-O 1.31.x** commonly cited; match vendor guidance.
3. **Feature gates:** Several beta features moved toward GA; check the changelog if you rely on gates.

## Pre-upgrade checks

```bash
# 1. Current cluster version
kubectl version --short

# 2. Nodes
kubectl get nodes -o wide

# 3. Pods
kubectl get pods -A

# 4. Deprecated API usage (metrics scrape)
kubectl get --raw /metrics | grep apiserver_requested_deprecated_apis

# 5. etcd backup — follow Upgrade procedure (linked at top)

# 6. Component health (prefer pod-based checks; componentstatus is deprecated)
kubectl get pods -n kube-system
```

## Upgrade process

### Step 1: Upgrade control plane (kubeadm)

```bash
# On the first control plane node

# 1. Refresh package metadata
sudo apt update   # Debian/Ubuntu
# OR
sudo yum update   # RHEL/CentOS

# 2. Inspect available kubeadm
sudo apt-cache madison kubeadm | grep 1.32
# OR
sudo yum list --showduplicates kubeadm --disableexcludes=kubernetes | grep 1.32

# 3. Install kubeadm v1.32.x (Debian/Ubuntu example)
sudo apt-mark unhold kubeadm && \
sudo apt-get update && \
sudo apt-get install -y kubeadm=1.32.0-00 && \
sudo apt-mark hold kubeadm

# OR RHEL/CentOS example:
# sudo yum install -y kubeadm-1.32.0-0 --disableexcludes=kubernetes

kubeadm version

# 4. Plan and apply (after new kubeadm is on disk)
sudo kubeadm upgrade plan
sudo kubeadm upgrade apply v1.32.0

# 5. Drain before kubelet/kubectl upgrade on this node (from a host with kubectl)
kubectl drain <control-plane-node> --ignore-daemonsets
```

#### Upgrade kubelet and kubectl on the control plane

```bash
sudo apt-mark unhold kubelet kubectl && \
sudo apt-get update && \
sudo apt-get install -y kubelet=1.32.0-00 kubectl=1.32.0-00 && \
sudo apt-mark hold kubelet kubectl

# OR RHEL/CentOS:
# sudo yum install -y kubelet-1.32.0-0 kubectl-1.32.0-0 --disableexcludes=kubernetes

sudo systemctl daemon-reload
sudo systemctl restart kubelet
kubectl uncordon <control-plane-node>
kubectl get nodes
```

#### Additional control plane nodes

Repeat on each member: install target **kubeadm**, then use **`sudo kubeadm upgrade node`** instead of **`kubeadm upgrade apply`**, then upgrade **kubelet**/**kubectl** and restart **kubelet** (with drain/uncordon as appropriate).

---

### Step 2: Upgrade worker nodes

Upgrade **one worker at a time**.

```bash
# From control plane / admin — drain
kubectl drain <worker-node-name> --ignore-daemonsets --delete-emptydir-data

# On the worker — kubeadm, then node config, then kubelet/kubectl
sudo apt-mark unhold kubeadm && \
sudo apt-get update && \
sudo apt-get install -y kubeadm=1.32.0-00 && \
sudo apt-mark hold kubeadm

sudo kubeadm upgrade node

sudo apt-mark unhold kubelet kubectl && \
sudo apt-get update && \
sudo apt-get install -y kubelet=1.32.0-00 kubectl=1.32.0-00 && \
sudo apt-mark hold kubelet kubectl

sudo systemctl daemon-reload
sudo systemctl restart kubelet

kubectl uncordon <worker-node-name>
kubectl get nodes
```

**RHEL/CentOS:** use `yum install` lines analogous to the control plane section.

Repeat for every worker.

kubectl get pods -A

## Rollback

Start with **[Upgrade procedure](./upgrade_procedure.md)** for etcd restore and package downgrade strategy. The steps below are an **illustrative** sequence; static-pod kubeadm clusters often differ (for example apiserver as a pod, not `systemctl kube-apiserver`).

```bash
sudo systemctl stop kube-apiserver

ETCDCTL_API=3 etcdctl snapshot restore /backup/etcd-snapshot-<timestamp>.db \
  --data-dir=/var/lib/etcd-restore

sudo mv /var/lib/etcd /var/lib/etcd.backup
sudo mv /var/lib/etcd-restore /var/lib/etcd

sudo systemctl restart etcd
sudo systemctl restart kube-apiserver

kubectl get nodes
kubectl get pods -A
```

## Troubleshooting

### Node stuck in NotReady

```bash
kubectl describe node <node-name>
sudo journalctl -u kubelet -n 100 --no-pager
sudo systemctl restart kubelet
```

### Pods not starting

```bash
kubectl describe pod <pod-name> -n <namespace>
sudo systemctl status containerd
# OR
sudo systemctl status crio
kubectl top nodes
kubectl top pods -A
```

### API server not responding

```bash
sudo journalctl -u kube-apiserver -n 100 --no-pager

ETCDCTL_API=3 etcdctl --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key \
  endpoint health

sudo systemctl restart kube-apiserver
```

## Summary

- Back up etcd and read the **v1.32** changelog before changing binaries.
- **Control plane first** (`kubeadm upgrade apply` then `upgrade node` on other masters).
- **Workers** one by one with **drain** / **uncordon**.
- Validate **nodes**, **kube-system**, **events**, and deprecated API metrics after the upgrade.

**Rough downtime:** on the order of minutes per control plane member (HA reduces user-visible impact) and per worker, depending on workload eviction and image pulls.
